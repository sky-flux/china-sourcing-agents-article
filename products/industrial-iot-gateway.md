An industrial IoT gateway is a ruggedized edge computer that sits between field devices and the cloud, translating legacy industrial protocols into modern message formats. It collects data from PLCs, sensors, meters, and machines over RS-485, Modbus, or Ethernet, then publishes that data to an MQTT broker, SCADA system, or cloud platform. Unlike a consumer router, it is built for electrical noise, wide temperature swings, and unattended operation in factories, pipelines, and remote assets.

The primary value of an industrial IoT edge gateway is translating legacy serial protocols (Modbus RTU/ASCII) to modern cloud protocols (MQTT, OPC-UA) for smart factory integration and predictive maintenance. When sourcing an industrial IoT gateway, evaluate the protocol bridging capabilities and edge computing power in detail:

**Modbus slave to MQTT publisher.** The gateway polls Modbus RTU devices (PLCs, sensors, meters) on the RS-485 bus and publishes register values to an MQTT broker at configurable intervals. Key parameters: polling rate (down to 100ms for fast data), register count per device, number of simultaneous Modbus slave devices supported (typical: 32–247 per bus segment).
